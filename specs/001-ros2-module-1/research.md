## 1. Docusaurus 3.x Setup

### Decision: Use Docusaurus Classic Template

**Installation Command**:
```bash
npx create-docusaurus@latest book classic
cd book
npm start
```

**Rationale**: The "classic" template includes standard documentation features, blog support, and dark mode out of the box.

**Alternatives Considered**:
- Bare template: Too minimal, requires more setup
- Custom template: Unnecessary complexity for documentation site

### GitHub Pages Deployment Configuration

**docusaurus.config.js**:
```javascript
const config = {
  url: 'https://<username>.github.io',
  baseUrl: '/Hackathon-book-creation-2025/',
  organizationName: '<username>',
  projectName: 'Hackathon-book-creation-2025',
  deploymentBranch: 'gh-pages',
  trailingSlash: false,
};
```

**GitHub Actions Workflow** (`.github/workflows/deploy.yml`):
```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 18
      - name: Install dependencies
        run: cd book && npm install
      - name: Build
        run: cd book && npm run build
      - uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./book/build
```

### Key DDS Concepts to Cover

1. **What is DDS**: Data Distribution Service - industry-standard middleware
2. **Publish-Subscribe Pattern**: Decoupled communication via topics
3. **Quality of Service (QoS)**: Reliability, durability, deadline policies
4. **Discovery**: Automatic node discovery (no roscore needed)
5. **RTPS Protocol**: Real-Time Publish-Subscribe wire protocol

**DDS Implementations in ROS 2 Humble**:
- Eclipse Cyclone DDS (default)
- eProsima Fast DDS
- RTI Connext DDS (commercial)

## 3. Docusaurus Content Structure

### Chapter Frontmatter Schema

```yaml
---
sidebar_position: 1
title: "Chapter Title"
description: "Brief description for SEO and link previews"
tags: [ros2, humanoid, tutorial]
---
```

**Required Fields**:
- `sidebar_position`: Numeric order in sidebar
- `title`: Display title (generates H1)

**Recommended Fields**:
- `description`: For SEO meta tags
- `tags`: For categorization and search

### Code Syntax Highlighting

## 4. Content Patterns

### Chapter Structure Pattern

Each chapter follows this structure:

```markdown
---
sidebar_position: N
title: "Chapter Title"
description: "Brief description"
---

## What You'll Learn

- Learning objective 1
- Learning objective 2

## Main Content

[Chapter content with code examples]

## Key Takeaways

- Takeaway 1
- Takeaway 2

## Next Steps

Link to next chapter or related resources
```

### Code Example Pattern

Every code block includes:
1. **Title**: File name or descriptive title
2. **Expected Output**: What reader should see
3. **Explanation**: Why the code works
